# Документация плагина Computer Use

Плагин `computer_use` позволяет агенту взаимодействовать с графическим интерфейсом операционной системы (эмуляция мыши, клавиатуры, создание скриншотов). Плагин поддерживает два различных архитектурных режима (бэкенда) работы: нативный Windows и изолированный графический сервер в WSL.

---

## Бэкенды и их отличия

### 1. `wsl` (По умолчанию)
Агент выполняет действия внутри Windows Subsystem for Linux (WSL), используя невидимый виртуальный дисплей (Xvfb).

* **Преимущества (Изоляция):** Агент работает в фоне и **не мешает вашей работе** на основном компьютере. Ваша реальная мышь и клавиатура остаются свободными.
* **Требования:** **ПЕРЕД ПЕРВЫМ ЗАПУСКОМ ТРЕБУЕТСЯ НАСТРОЙКА WSL**.
* **Как наблюдать:** Агент автоматически поднимает VNC-сервер. Вы можете наблюдать за процессом в реальном времени прямо из браузера, перейдя по ссылке: 
  👉 `http://127.0.0.1:6080/vnc.html`
* **Технологии:** `xdotool` (эмуляция ввода), `scrot` (скриншоты), `Xvfb` (виртуальный экран), `x11vnc` + `novnc` (трансляция).

### 2. `windows_base` (Нативный Windows)
Агент напрямую управляет вашей хост-системой Windows.

* **Преимущества (Реальная среда):** Агент видит ровно то же самое, что и вы на своем основном мониторе, и может взаимодействовать с вашими локальными приложениями. Также этот backend **НЕ ТРЕБУЕТ НАСТРОЙКИ**, что позволяет.
* **Ограничения:** Во время работы агента **нельзя трогать мышь и клавиатуру**, иначе вы собьете скрипт.
* **Индикация:** Для безопасности при активности агента на экране появляется оверлей поверх всех окон: `🤖 Агент управляет компьютером...`.
* **Технологии:** `pyautogui` (эмуляция ввода), `mss` (скриншоты), `tkinter` (оверлей).

## Как изменить бэкенд

Настройка бэкенда производится в конфигурационном файле `plugin_config.json`, который лежит в корне агента. 

Чтобы переключить бэкенд, добавьте или измените блок `settings -> computer_use`:

```json
{
  "settings": {
    "computer_use": {
      "backend": "windows_base" 
    }
  },
  "list": [
    "web_interface",
    "computer_use"
  ]
}
```
*(Допустимые значения `"backend"`: `"wsl"` или `"windows_base"`). После сохранения файла перезапустите агента.*

## Настройка окружения WSL

Если вы используете бэкенд `wsl` (рекомендуется), вам потребуется единоразово настроить окружение внутри дистрибутива Linux (например, Ubuntu), чтобы у агента был доступ к графическим библиотекам. 

Автоматизация старта (скрипт `start_vnc.sh`) уже предусмотрена плагином, но сами пакеты нужно установить.

Вы можете попросить агента установить и настроить wsl (в ./plugin_config.json нужно удалить `computer_use` из `list` при проблемах с запуском), либо настроить вручную:

Установите wsl, если не установлен: https://learn.microsoft.com/ru-ru/windows/wsl/install.

Откройте терминал WSL (`wsl` в PowerShell) и выполните установку зависимостей:

**1. Установка графического стека и утилит автоматизации:**
```bash
sudo apt update
sudo apt install -y xvfb fluxbox x11vnc novnc websockify xdotool scrot psmisc wmctrl
```

**2. Установка браузера Google Chrome (для навигации агента по вебу):**
```bash
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo apt install -y ./google-chrome-stable_current_amd64.deb
rm google-chrome-stable_current_amd64.deb
```

### Устранение частых проблем (Troubleshooting)
* **Черный экран в VNC / Ошибки кликов:** Убедитесь, что скрипт `start_vnc.sh` смог запуститься (агент запускает его сам при первом обращении, логи пишутся в консоль агента).
* **Не открывается `127.0.0.1:6080`:** Возможно, порты `6080` или `5900` заняты другой программой в Windows. Освободите порты или перезапустите WSL (`wsl --shutdown`).
