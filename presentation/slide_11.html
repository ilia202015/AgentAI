Взаимодействие с GUI: Браузер

Проблема "Слепоты"

Языковые модели отлично пишут скрипты, но абсолютно беспомощны при попытке выполнить простейшие задачи в Графическом Интерфейсе Пользователя (GUI), особенно на веб-сайтах.

Пути Решения

1
Передача сжатого DOM-дерева (Подходит для веба)
2
Мультимодальные модели (VLM) + Скриншоты

Реализация Подхода №1

Плагин Browser Use

Специализированное браузерное расширение извлекает из страницы текст и индексирует интерактивные элементы (кнопки, ссылки). Агент управляет страницей через инъекцию JavaScript по уникальным ID элементов.

Структура вывода browser_get_dom
"title": "Page", "url": "https://..."
"fullText": "Весь текст (вкл. IFRAME) для поиска данных"
"interactive": [
{ tag: "button", text: "Login", id: "btn-log", class: "btn" },
{ tag: "input", text: "", id: "user", class: "input" }
]
Только видимые элементы (скрытые фильтруются) ID — самый надежный способ для селектора
